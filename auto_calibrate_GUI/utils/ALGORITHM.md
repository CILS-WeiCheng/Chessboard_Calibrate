# 智能單目棋盤格挑選器 (Optimized Single Chessboard Selector) 演算法說明

本文件說明 `auto_picker.py` 中 `OptimizedSingleChessboardSelector` 類別的核心演算法流程與邏輯概念。

## 1. 核心目標
在進行單目相機校正（Camera Calibration）時，並非圖片越多越好。過多相似的圖片會增加計算負擔，且若圖片分佈不均，會導致相機內參（如焦距、畸變參數）估算偏差。本演算法旨在自動選出一個包含約 15-20 張影像的優質子集，具備：
- **完整空間覆蓋**：確保棋盤格出現在影像的各個角落（邊緣畸變修正）。
- **豐富的姿勢資訊**：涵蓋不同的傾斜角度，以精確估算焦距與主點。
- **極低的重投影誤差 (RMS)**：透過迭代篩選掉品質不佳或檢測不準的影像。

## 2. 演算法流程

### 第一階段：特徵提取與品質評分 (Feature Extraction & Scoring)
系統讀取目錄下所有影像，對每張影像執行：
1. **角點檢測**：尋找棋盤格角點並精化至次像素精度。
2. **幾何屬性分析**：
    - **九宮格分區 (Region)**：將影像分為 3x3 區域，記錄棋盤格中心所處位置。
    - **影像覆蓋率 (Coverage)**：棋盤格在畫面中佔的大小。
    - **傾斜角度 (Angle)**：棋盤格相對於影像平面的偏轉程度。
3. **動態加權評分**：
    - `Score = Coverage * (1.0 + Angle / 30.0)`
    - 此公式優先選擇「大面積」且「有傾斜角度」的影像，因為這類影像對參數估算的約束力最強。

### 第二階段：空間與多樣性初步挑選 (Initial Selection)
1. **區域保底**：優先從 9 個區域中各選出一張分數最高的圖片，確保空間上的「絕對覆蓋」。
2. **姿勢分群 (K-Means Clustering)**：
    - 針對剩餘的候選圖，根據「角度」與「縮放比」進行 K-Means 聚類。
    - 從每個類別中挑選代表性圖片。這能避免選到過多重複姿勢的圖片（例如雖然都在中心區域，但角度都一模一樣的連拍圖）。

### 第三階段：迭代誤差優化 (Iterative RMS Optimization)
這是確保校正精度達到亞像素級的核心步驟：
1. **試校正**：對目前選中的圖片進行試驗性校正，計算整體的 RMS 與「每張圖的個別投影誤差」。
2. **尋找最差圖片**：找出誤差貢獻度最高（最不準）的一張圖。
3. **區域一致性檢查 (Spatial Consistency)**：
    - 嘗試將該「爛圖」替換掉。
    - **限制條件**：如果該圖是該區域（3x3 Grid）的唯一一張，則替換時必須同樣從該區域選取候補，否則會破壞空間覆蓋率。
4. **擇優替換**：如果替換後整體 RMS 降低，則接受該次更動；否則嘗試下一個候選者。
5. 持續此循環直到 RMS < 0.5 px 或達到迭代次數上限。

### 第四階段：分析視覺化 (Visualization)
最後會產生 `single_selection_analysis.png`，包含：
- **空間分佈直方圖**：確認 3x3 區域是否都有取樣。
- **角度分佈圖**：顯示圖片姿態的豐富度。
- **評分相關性分析**：分析挑選出的圖片之品質分佈。

## 3. 邏輯概念總結

| 核心技術 | 解決問題 |
| :--- | :--- |
| **3x3 Grid Region** | 避免校正參數偏重於影像中心，解決邊緣拉伸/壓縮問題。 |
| **K-Means Clustering** | 解決資料冗餘問題，確保用最少的圖片涵蓋最多的姿勢。 |
| **RMS Feedback Loop** | 動態修正檢測誤差，將校正精度自動推向極限。 |
| **Angle Bonus** | 解決單一平面校正時焦距（focal length）估算不穩定的問題。 |
